WHAT WE ARE OFFERING?

Front Desk/Reception

Replying to customer queries about doctors' sessions and making appointments are two crucial functions in any hospital/channelling centre. This module cater for that requirement. In addition, this module covers essential functions such as patient registration, making appointments, admit patients to inwards, making payments for appointments, making payments for radiology & laboratory test, doctors' room management, and managing doctors' schedules.


OPD

Out Patient Department (OPD) is the most important module in this system. Maintaining Electronic Medical Record (EMR) of a registered patient is the main function of this module. Once patient is registered at the Front Desk/Reception, adding allergies and creating a visit, record patients clinical history, diagnosis and treatments, prescribing medicine, request laboratory/radiology reports, adding additional attachements to current observation are done in this module. Moreover, OPD queue management is done in this module.


Clinic

Modules to cover functions of individual specialized clinics are currently being developed. Peadiatric respiratory clinic and oncology clinic modules are available.


Lab

This module covers two basic areas, sample collection room (bleeding room) funcions and Laboratory functions. Managing sample collection, bar coded labeling, and maintaining bill of material (BoM) for inventory handling can be done using this module. Serching and filtering tests and reports, performance dashboard, creating new laboratory tests, parameterizing test feilds and make equations for said parameters, automatic/human data entering, generate reports individually or as bulks can be done in this module. Auditing function to track samples and staff involved with samples can be done using this module.


Pharmacy

This module has two roles the pharmacist, cashier. It can customize according to user requirement. If the module uses both user roles, the pharmacist can create a new bill request or assign patient bill request to their counter and do drug lock. Then cashier can generate a bill by searching patient. If the only cashier uses this module, the cashier can directly make a bill by searching doctor's bill request or as a new bill. The drug lock feature only works when using both user roles. This module depends on the inventory module.


In-ward

Monitoring admitted patients' is the aim of this module. Ward and room creation, adding facilities to each room and assign room charges, assigning meal plans can be done in this module. Nurse can add daily monitoring values of each patient and doctor can add treatmet plan and procedures for every patient. Moreover it can handle emergency treatments, meal plan, death, room transfer and discharge. Report/bill generation can be done at discharge, automatic bill generation for part payments, room allocation, treatment execution, procedure execution, lab requests, radiology requests, emergency treatments and for meals.


RIS

This module is used by the Radiology Department of the Hospital and can be used to track the progress of a radiology exam from doctor's request to radiologist's report. This module has a dedicated PACS server, a web-based image (DICOM) viewer and a feature-rich stand-alone image (DICOM) viewer. This can recieve radiology requests from OPD and in-ward doctors, or can create a radiology request according to a request from an external doctor.


HR

Department information, employee management, designation, pay slips and user credential creates are part HR module.


Billing/Inventory

Billing uses in generating service billing. The invoice can generate by either bill requests which send by a doctor or as a new bill by search services which are added by charge master module. If cashier bill lab test or radiology test, test requests send to lab or radiology module accordingly. Here also cashier can handle advance payment, credit payment and refunds.
